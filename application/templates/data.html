Data API

New data fetched at the start of each hour

  Our data API consists of two types of data: COVID-19 spread data (source) and state-issued reopening order dates (source). As such, our API is divided into two main sections to allow access to both types of data. When using this API, we kindly ask that you credit The Atlantic's COVID Tracking Project, the New York Times, and our own work here as well.
  Our datasets differ from all current existing ones for two main reasons. First, our novel algorithm can convert current to cumulative hospitalizations with a low error rate This ensures that all states have values for cumulative hospitalizations, which is incredibly useful for calculating doubling time and other important spread metrics. Furthermore, our reopening order database is the first of its kind for state-issued orders, providing a streamlined data process for future works with easy-to-use categorizations.

COVID-19 Spread Data

/data/api/covid: Get all COVID-19 data for all states (cases, tests, hospitalziations, deaths, and more).

/data/api/covid/{state}: Get all COVID-19 data for one state (cases, tests, hospitalziations, deaths, etc.).

  Possible Options for {state}:

  • Two-letter state abbreviation (ex: fl, tx, ny, dc)


  Example: /data/api/covid/fl

/data/api/covid/{field}: Get one type of COVID-19 data for all states.

  Possible Options for {field}:

  • positive (total number of cases)
  • negative (total number of negative tests)
  • pending (number of pending tests)
  • hospitalizedCurrently (number of currently hospitalized patients)
  • hospitalizedCumulative (total number of hospitalized patients)
  • death (total number of deaths)
  • recovered (total number of recovered patients)


  Example: /data/api/covid/positive

/data/api/covid/{state}/{field}: Get one type of COVID-19 data for one state.

State-Issued Reopening Order Data

/data/api/reopening: Get state-issued reopening order dates for all states and all categories.

/data/api/reopening/{state}: Get state-issued reopening order dates for one state, all categories.

  Possible Options for {state}:

  • Two-letter state abbreviation (ex: fl, tx, ny, dc)


  Example: /data/api/reopening/fl

/data/api/reopening/{type}:

  Possible Options for {type}:

  • restaurants_and_bars (Indoor Dining with Bars)
  • restaurants (Indoor Dining without Bars)
  • restaurants_outdoors (Outdoor Dining)
  • limited_retail (Curbside Pickup Retail)
  • retail (Indoor Retail)
  • gyms (Gyms and Fitness Centers)
  • entertainment (Indoor Entertainment)
  • hair_salons (Hair Salons and Barbershops)
  • personal_care (Non-Hair Personal Care)
  • office (Office Environments)
  • houses_of_worship (Houses of Worship)
  • construction (Construction)
  • beaches (Beaches)
  • parks (Campgrounds and State Parks)


  Example: /data/api/reopening/restaurants

/data/api/reopening/{state}/{type}: Get state-issued reopening order dates for one state, one category.

  Example: /data/api/reopening/restaurants
